# Minimal Design Template

Use this template by default.

Add optional sections only when they change a review decision.

```markdown
# <title>

## Objective

- What problem is being solved
- What outcome is expected

## Scope

- In scope
- Out of scope

## Context

- Impacted components
- Key stakeholders
- Relevant existing behavior

## Requirements And Assumptions

- Functional requirements
- Non-functional requirements
- Assumptions
- Open questions

## Proposed Design

- Main solution shape
- Key design decisions
- Interfaces and dependencies

## Risks And Failure Handling

- Main technical risks
- Error scenarios
- Mitigations

## Verification Strategy

- Unit tests
- Integration tests
- Manual verification

## Decision Needed

- What needs review, approval, or alignment
```

## Optional Sections

Add these only when they matter:

- Diagrams
- Performance
- Security
- Maintainability
- Alternative solutions considered
- Planning
- References
