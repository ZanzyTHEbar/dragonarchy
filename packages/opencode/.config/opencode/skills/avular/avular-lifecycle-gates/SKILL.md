# Avular Lifecycle Gates

Use this skill when engineering work is tied to a product lifecycle stage or gate decision.

## Workflow

1. Identify the current stage and the next gate.
2. Determine what engineering evidence is needed for that stage.
3. Check which technical deliverables are complete, incomplete, or waived.
4. Summarize residual risk and open questions.
5. Recommend proceed, proceed with waiver, postpone, or re-scope.

## Guardrails

- Focus on stage readiness, not full project management.
- Bring only the engineering evidence needed for the current decision.
- A waiver is acceptable only when the residual risk is explicit.
- If the work does not affect a lifecycle transition, do not force gate language into the task.

## Output

- Current stage and next gate
- Missing engineering evidence
- Residual technical risks
- Recommended gate posture
