# Stage Summary

Use this as a software-engineering view of the lifecycle.

## Definition

Goal:

- decide whether the opportunity is desirable, viable, feasible, and aligned

Engineering focus:

- initial feasibility
- high-level technical risks
- rough solution direction

## Preparation

Goal:

- prepare development with a clear technical basis

Engineering focus:

- technical requirements
- initial system architecture
- design documentation
- development and verification approach

## Development

Goal:

- build and integrate the product through iterative maturation

Engineering focus:

- implementation
- integration
- prototype learning
- verification and validation progress

## User Test And Optimize

Goal:

- validate in realistic use and prepare for production readiness

Engineering focus:

- bug fixing
- usability and workflow optimization
- production-readiness issues
- remaining compliance-related technical work

## Maintenance And Support

Goal:

- support deployed products and keep them operational

Engineering focus:

- robustness
- updates
- monitoring
- supportability
- long-term maintainability

## Product Phase-Out

Goal:

- support customers while retiring the product cleanly

Engineering focus:

- service-only support
- essential updates only
- decommissioning impacts
