# Avular Product Versioning

Use this skill when the work depends on Avular's product hierarchy or version language.

## Workflow

1. Identify whether the subject is Tier-0, Tier-1, Tier-2, or Tier-3.
2. Apply the correct version format for that tier.
3. Describe the change using the tier's actual compatibility semantics.
4. Keep internal language consistent across design, release, and architecture documents.

## Guardrails

- Do not collapse tiers into generic semantic-version language.
- Tier language matters for architecture and release communication.
- Use examples when naming is ambiguous.

## Output

- The correct tier label
- The correct version format
- The correct wording for compatibility or release impact
