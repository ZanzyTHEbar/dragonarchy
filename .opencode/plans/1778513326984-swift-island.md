# Plan: Ansible Local Connection for Self-Targeted Runs

## Problem

The current Ansible implementation defaults to the `smart` connection plugin, which falls back to SSH for all inventory hosts. When `run-convergence.sh --host firedragon` is executed on `firedragon` itself, Ansible attempts to SSH into `firedragon` (which resolves to `127.0.0.1`). While this can work if SSH keys and host entries are configured, it is unnecessary overhead, slower, and introduces a failure mode (SSH agent, keys, daemon) for what should be a direct local execution.

The user wants Ansible to execute directly on the local machine when the target is the host machine itself, while preserving SSH for remote execution (hybrid model: sometimes remote, sometimes local).

## How Ansible Connection Works

- **Default behavior**: Without explicit configuration, Ansible's `smart` connection plugin chooses `ssh` for all non-`localhost` inventory hosts.
- **`--connection=local`**: Forces Ansible to execute modules directly on the control node (the machine running `ansible-playbook`) without opening an SSH session. The host is still processed with its full inventory variables (`inventory_hostname`, host vars, group vars).
- **`delegate_to: localhost`**: Changes the execution target for a specific task to `localhost`, which uses the `localhost` inventory context (losing target host variables). This is wrong for entire playbooks.
- **Inventory-level `ansible_connection`**: Setting this in `host_vars` or inventory would hardcode the connection for that host, breaking remote execution from a control node. Not suitable for a hybrid model.

## Design Decision

**Approach**: Add a thin wrapper script that auto-detects when `--limit` matches the current machine and injects `--connection=local`. Update `run-convergence.sh` to use this wrapper, and update runbooks to document the new pattern.

**Rationale**:
- **No inventory changes**: Avoids breaking remote execution from a control node.
- **Minimal intrusion**: Only the execution layer changes; playbooks and roles are untouched.
- **Explicit and observable**: The wrapper logs when it switches to local connection.
- **Preserves semantics**: `--connection=local` keeps the target host's identity and variables intact (unlike `delegate_to`).

### Why not SSH-to-self?
- Works today (hostnames resolve to localhost), but adds connection overhead, requires SSH daemon + key auth, and is a needless failure surface for local-only runs.

### Why not `ansible_connection=local` in inventory?
- Would permanently disable SSH for that host, breaking the "sometimes remote" use case.

### Why not `delegate_to: localhost`?
- Loses host-specific variables and inventory context. Suitable for single tasks, not whole playbooks.

## Implementation

### 1. Create `infra/ansible/run-playbook.sh`

A wrapper around `ansible-playbook` that:
- Accepts the same arguments as `ansible-playbook`
- Parses `--limit <host>`
- Detects if `<host>` is the current machine by comparing against `hostname -s` and `hostname -f`
- If local, prepends `--connection=local` and logs the decision
- Otherwise, passes through unchanged

Key logic:
```bash
is_local_target() {
    local target="$1"
    local current_short current_fqdn
    current_short=$(hostname -s 2>/dev/null || hostname)
    current_fqdn=$(hostname -f 2>/dev/null || hostname)

    [[ "${target}" == "localhost" ]] || \
    [[ "${target}" == "127.0.0.1" ]] || \
    [[ "${target}" == "::1" ]] || \
    [[ "${target}" == "${current_short}" ]] || \
    [[ "${target}" == "${current_fqdn}" ]]
}
```

### 2. Modify `infra/run-convergence.sh`

Replace direct `ansible-playbook` invocations with calls to the wrapper. The wrapper handles `--connection=local` injection only when `--limit` is present and matches the local host.

**Important**: `foundation.yml` targets `hosts: all` and is invoked **without** `--limit`. Therefore, it will **not** receive `--connection=local` and will continue to use SSH (default). This preserves its "global contract check" semantics. If running locally without SSH to all peers, the operator can still run it with `--limit` manually.

### 3. Update documentation

Update all manual invocation examples to use the wrapper or show the `--connection=local` flag for local runs.

Files to update:
- `docs/runbooks/first-host-chezmoi-cutover.md`
- `hosts/firedragon/README.md`

### 4. Verification

- **Test local execution**: Run `infra/run-convergence.sh --host $(hostname -s) --dry-run` and verify in the output that `--connection=local` is passed to `site.yml` and that no SSH connection is attempted.
- **Test remote execution semantics**: Confirm that when `--host` does not match the current machine, no `--connection=local` is added.
- **Test manual wrapper**: Run `infra/ansible/run-playbook.sh playbooks/site.yml --limit $(hostname -s) --check` and verify local execution.
- **Test foundation global check**: Confirm `foundation.yml` still runs on `all` hosts without `--connection=local`.
